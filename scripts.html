Google Earth Engine Data Pull

Below I include the JavaScript I wrote to extract annual geospatial indicators for Sweden at the county level. The script (updated May 11, 2025) defines the target year (2024) with fallback to 2023, loads and simplifies the county boundaries, and then for each county calculates:

  • Nighttime lights (VIIRS avg radiance): monthly VIIRS DNB images averaged annually.

  • Urban cover: Dynamic World built-area pixels (class 6) mode, converted to percent.

  • NDVI: MODIS Terra NDVI mean, scaled.

  • Land surface temperature: MODIS LST mean, converted to °C.

  • Precipitation: CHIRPS daily sum.

  • Tropospheric NO₂: Sentinel-5P NO₂ column mean.

  • Soil moisture: GLDAS top-10 cm moisture mean.

  • Elevation & slope: Copernicus GLO-30 DEM and derived slope.

It concatenates all these bands, computes per-county means at 1 km resolution, and finally exports the selected variables as CSV to Google Drive.

// ====================================================================
// ANNUAL GEOINDICATORS FOR SWEDEN (COUNTY LEVEL) – 2024
// Computes yearly averages for 2024
// Updated : 2025-05-11
// ====================================================================

// PARAMETERS
var targetYear   = '2024',
    fallbackYear = '2023',
    startDate    = targetYear   + '-01-01',
    endDate      = targetYear   + '-12-31',
    fbStart      = fallbackYear + '-01-01',
    fbEnd        = fallbackYear + '-12-31';

// 0. ADMINISTRATIVE UNITS
var counties = ee.FeatureCollection('projects/geodata-458113/assets/SWE_adm1');
var simplifiedCounties = counties.map(function(f) {
  return f.simplify({maxError: 100});
});

// ====================================================================
// 1. INDICATORS – YEARLY AVERAGES
// ====================================================================

// NIGHTTIME LIGHTS (VIIRS DNB Monthly)
// Dataset: NOAA/VIIRS/DNB/MONTHLY_V1/VCMCFG
// Definition: Average radiance (avg_rad)
// Purpose: Proxy for human activity & economic development
// Period: Jan 1 – Dec 31 (annual)
// Resolution: ~500 m
// More info: https://developers.google.com/earth-engine/datasets/catalog/NOAA_VIIRS_DNB_MONTHLY_V1_VCMCFG
var viirsCol = ee.ImageCollection('NOAA/VIIRS/DNB/MONTHLY_V1/VCMCFG')
                 .filterDate(startDate, endDate)
                 .select('avg_rad');
var viirsAvail = viirsCol.size().gt(0);
var viirsFallbackCol = ee.ImageCollection('NOAA/VIIRS/DNB/MONTHLY_V1/VCMCFG')
                           .filterDate(fbStart, fbEnd)
                           .select('avg_rad');
var viirsImg = ee.Image(ee.Algorithms.If(
                 viirsAvail,
                 viirsCol.mean(),
                 viirsFallbackCol.mean()
               )).rename('VIIRS_avg_2024');

// URBAN COVER (Dynamic World)
// Dataset: GOOGLE/DYNAMICWORLD/V1
// Definition: Pixel-level classification; class 6 = built area
// Purpose: Proxy for built-up extent
// Period: Jan 1 – Dec 31 (annual)
// Resolution: 10 m
// More info: https://developers.google.com/earth-engine/datasets/catalog/GOOGLE_DYNAMICWORLD_V1
var dwCol = ee.ImageCollection('GOOGLE/DYNAMICWORLD/V1')
              .filterDate(startDate, endDate)
              .select('label');
var dwAvail = dwCol.size().gt(0);
var dwFallbackCol = ee.ImageCollection('GOOGLE/DYNAMICWORLD/V1')
                     .filterDate(fbStart, fbEnd)
                     .select('label');
var urbanImg = ee.Image(ee.Algorithms.If(
                  dwAvail,
                  dwCol.mode().eq(6),
                  dwFallbackCol.mode().eq(6)
                )).multiply(100)
                  .rename('Urban_pct_2024');

// NDVI (MODIS Terra)
// Dataset: MODIS/006/MOD13A2
// Definition: NDVI at 1 km; scale factor ×0.0001
// Purpose: Vegetation greenness proxy
// Period: Jan 1 – Dec 31 (annual)
// Resolution: 1 km
// More info: https://developers.google.com/earth-engine/datasets/catalog/MODIS_006_MOD13A2
var ndviCol = ee.ImageCollection('MODIS/006/MOD13A2')
               .filterDate(startDate, endDate)
               .select('NDVI');
var ndviAvail = ndviCol.size().gt(0);
var ndviFallbackCol = ee.ImageCollection('MODIS/006/MOD13A2')
                      .filterDate(fbStart, fbEnd)
                      .select('NDVI');
var ndviImg = ee.Image(ee.Algorithms.If(
                 ndviAvail,
                 ndviCol.mean(),
                 ndviFallbackCol.mean()
               )).multiply(0.0001)
                 .rename('NDVI_avg_2024');

// LAND-SURFACE TEMPERATURE (MODIS)
// Dataset: MODIS/061/MOD11A2
// Definition: 8-day mean daytime LST at 1 km; scale ×0.02; convert to °C by subtracting 273.15
// Purpose: Thermal anomaly proxy
// Period: Jan 1 – Dec 31 (annual)
// Resolution: 1 km
// More info: https://developers.google.com/earth-engine/datasets/catalog/MODIS_061_MOD11A2
var lstCol = ee.ImageCollection('MODIS/061/MOD11A2')
              .filterDate(startDate, endDate)
              .select('LST_Day_1km');
var lstAvail = lstCol.size().gt(0);
var lstFallbackCol = ee.ImageCollection('MODIS/061/MOD11A2')
                     .filterDate(fbStart, fbEnd)
                     .select('LST_Day_1km');
var lstImg = ee.Image(ee.Algorithms.If(
                lstAvail,
                lstCol.mean(),
                lstFallbackCol.mean()
              )).multiply(0.02)
                .subtract(273.15)
                .rename('LST_C_2024');

// PRECIPITATION TOTAL (CHIRPS)
// Dataset: UCSB-CHG/CHIRPS/DAILY
// Definition: Daily precipitation sum (mm)
// Purpose: Wet/dry anomaly proxy
// Period: Jan 1 – Dec 31 (annual)
// Resolution: ~5 km
// More info: https://developers.google.com/earth-engine/datasets/catalog/UCSB_CHG_CHIRPS_DAILY
var prcpCol = ee.ImageCollection('UCSB-CHG/CHIRPS/DAILY')
               .filterDate(startDate, endDate);
var prcpAvail = prcpCol.size().gt(0);
var prcpFallbackCol = ee.ImageCollection('UCSB-CHG/CHIRPS/DAILY')
                      .filterDate(fbStart, fbEnd);
var prcpImg = ee.Image(ee.Algorithms.If(
                prcpAvail,
                prcpCol.sum(),
                prcpFallbackCol.sum()
              )).rename('Precip_mm_2024');

// TROPOSPHERIC NO₂ (Sentinel-5P)
// Dataset: COPERNICUS/S5P/OFFL/L3_NO2
// Definition: Tropospheric NO₂ column density (µmol/m²)
// Purpose: Emissions proxy
// Period: Jan 1 – Dec 31 (annual)
// Resolution: ~7 km
// More info: https://developers.google.com/earth-engine/datasets/catalog/COPERNICUS_S5P_OFFL_L3_NO2
var no2Col = ee.ImageCollection('COPERNICUS/S5P/OFFL/L3_NO2')
              .filterDate(startDate, endDate)
              .select('tropospheric_NO2_column_number_density');
var no2Avail = no2Col.size().gt(0);
var no2FallbackCol = ee.ImageCollection('COPERNICUS/S5P/OFFL/L3_NO2')
                      .filterDate(fbStart, fbEnd)
                      .select('tropospheric_NO2_column_number_density');
var no2Img = ee.Image(ee.Algorithms.If(
                no2Avail,
                no2Col.mean(),
                no2FallbackCol.mean()
              )).rename('NO2_mol_m2_2024');

// SOIL-MOISTURE (GLDAS-2.1 NOAH)
// Dataset: NASA/GLDAS/V021/NOAH/G025/T3H
// Definition: Soil moisture top 10 cm (SoilMoi0_10cm_inst)
// Purpose: Surface soil-water proxy
// Period: Jan 1 – Dec 31 (annual)
// Resolution: ~25 km
// More info: https://developers.google.com/earth-engine/datasets/catalog/NASA_GLDAS_V021_NOAH_G025_T3H
var smCol = ee.ImageCollection('NASA/GLDAS/V021/NOAH/G025/T3H')
              .filterDate(startDate, endDate)
              .select('SoilMoi0_10cm_inst');
var smAvail = smCol.size().gt(0);
var smFallbackCol = ee.ImageCollection('NASA/GLDAS/V021/NOAH/G025/T3H')
                      .filterDate(fbStart, fbEnd)
                      .select('SoilMoi0_10cm_inst');
var smImg = ee.Image(ee.Algorithms.If(
              smAvail,
              smCol.mean(),
              smFallbackCol.mean()
            )).rename('SoilM_m3m3_2024');

// ELEVATION & SLOPE (Copernicus GLO-30 Latest)
// Dataset: COPERNICUS/DEM/GLO30
// Definition: Digital elevation model; slope derived from DEM
// Purpose: Terrain ruggedness proxy
// Epoch: 2024 (latest mosaic)
// Resolution: 30 m
// More info: https://developers.google.com/earth-engine/datasets/catalog/COPERNICUS_DEM_GLO30
var demImg = ee.ImageCollection('COPERNICUS/DEM/GLO30')
              .select('DEM')
              .mosaic()
              .rename('Elevation_m_2024');
var slopeImg = ee.Terrain.slope(demImg)
                .rename('Slope_deg_2024');

// ====================================================================
// 2. COMBINE & REDUCE
// ====================================================================
var multi = ee.Image.cat([
  viirsImg, urbanImg, ndviImg,
  lstImg, prcpImg, no2Img, smImg,
  demImg, slopeImg
]);
var stats = multi.reduceRegions({
  collection: simplifiedCounties,
  reducer: ee.Reducer.mean(),
  scale: 1000,
  crs: 'EPSG:4326'
});

// ====================================================================
// 3. EXPORT – SELECTED VARIABLES ONLY
// ====================================================================
// Map to new features containing only County + mean indicators (renaming fields)
var exportCol = stats.map(function(f) {
  return ee.Feature(null, {
    'County': f.get('NAME_1'),
    'VIIRS_avg_2024': f.get('VIIRS_avg_2024_mean'),
    'Urban_pct_2024': f.get('Urban_pct_2024_mean'),
    'NDVI_avg_2024': f.get('NDVI_avg_2024_mean'),
    'LST_C_2024': f.get('LST_C_2024_mean'),
    'Precip_mm_2024': f.get('Precip_mm_2024_mean'),
    'NO2_mol_m2_2024': f.get('NO2_mol_m2_2024_mean'),
    'SoilM_m3m3_2024': f.get('SoilM_m3m3_2024_mean'),
    'Elevation_m_2024': f.get('Elevation_m_2024_mean'),
    'Slope_deg_2024': f.get('Slope_deg_2024_mean')
  });
});

Export.table.toDrive({
  collection : exportCol,
  description: 'Sweden_Annual_Geoindicators_2024_Selected',
  folder     : 'unsae',
  fileFormat : 'CSV'
});

SCB Data Pull

I query the Swedish Statistical Authority’s (SCB) API to retrieve:

Direct estimates (unemployment Q1 2025)

This Python script reads a JSON configuration (directEstimate.json) containing the API payload and table ID, builds the POST URL, fetches a CSV response, selects and renames critical columns (County, Percent_2025K1, Percent_2025K1_me), cleans the region names by stripping codes and suffixes, and writes the output to data/direct_estimates.csv.

import json
import requests
import pandas as pd
from io import StringIO
from pathlib import Path

# ------------------- 1. Load the Query Object and Table ID -------------------

config_path = Path("data/directEstimate.json")

with config_path.open("r", encoding="utf-8") as f:
    cfg = json.load(f)

payload = cfg["queryObj"]
table_id = cfg["tableIdForQuery"]

# ------------------- 2. Build the API URL -------------------

url = f"https://api.scb.se/OV0104/v1/doris/en/ssd/START/AM/AM0401/AM0401N/{table_id}"
headers = {"Content-Type": "application/json"}

# ------------------- 3. Fetch the Data -------------------

r = requests.post(url, json=payload, headers=headers)
r.raise_for_status()  # Raise an error if the request failed

# ------------------- 4. Parse According to Response Format -------------------

fmt = payload.get("response", {}).get("format", "").lower()

# Make sure 'data/' directory exists
data_dir = Path("data")
data_dir.mkdir(parents=True, exist_ok=True)

if fmt == "csv":
    # If the response is CSV format, read it into a DataFrame
    df = pd.read_csv(StringIO(r.text), sep=",")

    # Pick and rename the important columns
    df_selected = df.rename(columns={
        "region": "County",
        "Percent 2025K1": "Percent_2025K1",
        "Margin of error ± percent 2025K1": "Percent_2025K1_me"
    })[["County", "Percent_2025K1", "Percent_2025K1_me"]]

    # Clean the 'County' names: remove leading numbers and "county" word
    df_selected["County"] = (
        df_selected["County"]
        .str.replace(r"^\d+\s+", "", regex=True)     # Remove leading numbers and spaces
        .str.replace(r"\scounty$", "", regex=True)   # Remove " county" at the end
        .str.strip()                                 # Remove any leading/trailing whitespace
    )

    # Save the cleaned DataFrame to CSV
    output_filename = "direct_estimates.csv"
    out_csv = data_dir / output_filename
    df_selected.to_csv(out_csv, index=False)
    print(f"✅ Data saved to {out_csv.resolve()}")

elif fmt == "px":
    # If the response is PX format, save it as a PX file
    output_filename = f"{table_id}.px"
    out_px = data_dir / output_filename
    out_px.write_text(r.text, encoding="utf-8")
    print(f"✅ PC-Axis file saved to {out_px.resolve()}")

    # (Optional) Code example to read PX file later:
    # import pxpy
    # table = pxpy.read_pxd(str(out_px))
    # df = table.to_dataframe()
    # print(df.head())

else:
    # If the format is unknown, print the raw response
    print("⚠️ Response format not recognized. Here is the raw text:")
    print(r.text)

Population Density (2024)

This script mirrors the direct‐estimate process but targets the population density table. It loads popdensity.json for the query, pulls the CSV, renames to PopDensity_2024, standardizes county names, and writes to data/popdensity.csv.

import json
import requests
import pandas as pd
import re
from io import StringIO
from pathlib import Path

# 1. Load the query object and table ID from disk
config_path = Path("data/popdensity.json")
with config_path.open("r", encoding="utf-8") as f:
    cfg = json.load(f)

payload = cfg["queryObj"]
table_id = cfg["tableIdForQuery"]

# 2. Build the URL dynamically
url = f"https://api.scb.se/OV0104/v1/doris/en/ssd/START/BE/BE0101/BE0101C/{table_id}"

headers = {"Content-Type": "application/json"}

# 3. Fetch the data
r = requests.post(url, json=payload, headers=headers)
r.raise_for_status()

# 4. Parse according to response format
fmt = payload.get("response", {}).get("format", "").lower()

# 5. Handle CSV response
if fmt == "csv":
    df = pd.read_csv(StringIO(r.text), sep=",")

    # Pick and rename columns appropriately
    df_selected = df.rename(columns={
        "region": "County",
        "Population density per sq. km 2024": "PopDensity_2024"
    })[["County", "PopDensity_2024"]]

    # Clean 'County' names
    df_selected["County"] = (
        df_selected["County"]
        .str.replace(r"^\d+\s+", "", regex=True)     # remove leading numbers
        .str.replace(r"\scounty$", "", regex=True, flags=re.IGNORECASE)   # remove "county" at end
        .str.strip()
    )

    # Ensure 'data/' directory exists
    data_dir = Path("data")
    data_dir.mkdir(parents=True, exist_ok=True)

    # Save the cleaned DataFrame
    output_filename = "popdensity.csv"
    out_csv = data_dir / output_filename
    df_selected.to_csv(out_csv, index=False)
    print(f"Data saved to {out_csv.resolve()}")

elif fmt == "px":
    data_dir = Path("data")
    data_dir.mkdir(parents=True, exist_ok=True)

    output_filename = f"{table_id}.px"
    out_px = data_dir / output_filename
    out_px.write_text(r.text, encoding="utf-8")
    print(f"PC-Axis file saved to {out_px.resolve()}")

else:
    print("Response format not recognized. Here's the raw text:")

Load Library script

1_load_libraries.R ensures all required packages (sf, tidyverse, emdi, tmap, glmnet, here, conflicted) are installed and loaded, resolves function conflicts (e.g., filter and lag from dplyr), sets a reproducible seed, and defines a data_path() helper for consistent project-relative file access.

# 1_load_libraries.R

# ---------------------------------------------------------

# ---------------------------------------------------------
# Vector of packages needed for this project
target_pkgs <- c(
  "sf",        # spatial data handling
  "tidyverse", # data manipulation & visualization
  "emdi",      # small area estimation
  "tmap",      # thematic mapping
  "glmnet",    # regularized regression
  "here",      # project-relative file paths
  "conflicted" # manage function conflicts
)

# ---------------------------------------------------------
# 3. Install Missing Packages
# ---------------------------------------------------------
# Install any packages not already present
toinstall <- setdiff(target_pkgs, rownames(installed.packages()))
if (length(toinstall) > 0) {
  message("Installing missing packages: ", paste(toinstall, collapse = ", "))
  install.packages(toinstall)
}

# ---------------------------------------------------------

Clean Data script

2_sweden_preprocess.R:

  1. Sources the library loader.

  2. Defines file paths for shapefiles, direct estimates, geospatial data, population density, and job vacancies.

  3. Recodes county names (e.g., Örebro).

  4. Reads, cleans, and transforms each dataset:

    • Spatial shapefile → sf object in WGS84.

    • Direct estimates → numeric rates, variances, effective sample sizes.

    • Covariates (NDVI, urban cover, VIIRS nightlights).

    • Population density → numeric per km².

    • Latest vacancies → filters to most recent period.

  5. Merges all into a single combined_data table and performs a spatial join with the shapefile to create sweden_map_data.

  6. Saves processed data for downstream analysis.

# ---------------------------------------------------------
# Sweden Preprocess
# ---------------------------------------------------------

# ---------------------------------------------------------
# 1. Source Library Loader
# ---------------------------------------------------------
source(here::here("R", "1_load_libraries.R"))

# ---------------------------------------------------------
# 2. Define Paths & Helper Functions
# ---------------------------------------------------------
paths <- list(
  shapefile   = data_path("SWE_adm", "SWE_adm1.shp"),
  direct_est  = data_path("direct_estimates.csv"),
  geodata     = data_path("geodata2.csv"),
  popdensity  = data_path("popdensity.csv"),
  vacancies   = data_path("vacancies.csv")
)

# Consistent county recoding
name_map <- c("Orebro" = "Örebro")
recode_county <- function(x) {
  factor(dplyr::recode(as.character(x), !!!name_map))
}

# Wrapper for CSV imports with progress messages
read_data <- function(path) {
  message("Reading: ", path)
  readr::read_csv(path, show_col_types = FALSE)
}

# ---------------------------------------------------------
# 3. Load & Clean County Shapefile
# ---------------------------------------------------------
sweden_shape <-
  sf::st_read(paths$shapefile, quiet = TRUE) %>%
  sf::st_make_valid() %>%
  sf::st_transform(4326) %>%
  dplyr::select(-c(ID_0, ISO, NAME_0, TYPE_1, ENGTYPE_1, NL_NAME_1, VARNAME_1)) %>%
  dplyr::mutate(NAME_1 = recode_county(NAME_1)) %>%
  dplyr::arrange(NAME_1)
message("Shapefile loaded: ", nrow(sweden_shape), " polygons")

# ---------------------------------------------------------
# 4. Read Direct Estimates & Compute Variance
# ---------------------------------------------------------
direct_est <-
  read_data(paths$direct_est) %>%
  dplyr::mutate(
    County               = recode_county(County),
    Percent_2025K1       = na_if(Percent_2025K1, "..") %>% as.numeric() / 100,
    SE_95                = na_if(Percent_2025K1_me, "..") %>% as.numeric() / 100,
    standard_error       = SE_95 / 1.96,
    var_est              = standard_error^2,
    effective_sample_size= (Percent_2025K1 / standard_error)^2
  )
message("Direct estimates processed: ", nrow(direct_est), " records")

# ---------------------------------------------------------
# 5. Read & Clean Covariates
# ---------------------------------------------------------
# a) Geospatial covariates
geo_data <-
  read_data(paths$geodata) %>%
  dplyr::rename(
    NDVI_avg       = NDVI_avg_2024,
    UrbanCover_pct = Urban_pct_2024,
    VIIRS_avg      = VIIRS_avg_2024
  ) %>%
  dplyr::mutate(
    across(-NAME_1, as.numeric),
    NAME_1 = recode_county(NAME_1)
  ) 
message("Geodata loaded: ", nrow(geo_data), " rows")

# b) Population density
pop_density <-
  read_data(paths$popdensity) %>%
  dplyr::mutate(
    County          = recode_county(County),
    PopDensity_2024 = as.numeric(PopDensity_2024)
  )
message("Population density loaded: ", nrow(pop_density), " rows")

# c) Vacancies (latest period only)
vacancies <-
  read_data(paths$vacancies) %>%
  dplyr::filter(Period == max(Period, na.rm = TRUE)) %>%
  dplyr::mutate(
    County     = stringr::str_remove(Län, " län$"),
    County     = stringr::str_remove(County, "s$"),
    County     = recode_county(stringr::str_to_title(County)),
    Vacancy_New= as.numeric(`Nya lediga jobb`)
  ) %>%
  dplyr::select(County, Vacancy_New)
message("Vacancies loaded: ", nrow(vacancies), " rows")

# ---------------------------------------------------------
# 6. Merge All Data
# ---------------------------------------------------------
northern_list <- c("Norrbotten", "Västerbotten", "Jämtland", "Västernorrland", "Gävleborg")
combined_data <-
  direct_est %>%
  dplyr::left_join(geo_data,   by = c("County" = "NAME_1")) %>%
  dplyr::left_join(pop_density,by = "County") %>%
  dplyr::left_join(vacancies,  by = "County") %>%
  dplyr::mutate(
    Northern = factor(
      ifelse(as.character(County) %in% northern_list, "North", "South"),
      levels = c("South", "North")
    )
  )
# Validate merge
if (nrow(combined_data) != nrow(direct_est)) {
  stop("Merge error: row mismatch (", nrow(combined_data), " vs ", nrow(direct_est), ")")
}
message("Data merged: ", nrow(combined_data), " rows, all covariates included")

# ---------------------------------------------------------
# 7. Prepare Spatial Join for Mapping
# ---------------------------------------------------------
sweden_map_data <-
  sweden_shape %>%
  dplyr::left_join(combined_data, by = c("NAME_1" = "County"))
# Check for missing entries
missing_count <- sum(is.na(sweden_map_data$Percent_2025K1))
if (missing_count > 0) {
  warning(missing_count, " features missing direct estimates after join")
}
message("Spatial join complete: ", nrow(sweden_map_data), " features")

# ---------------------------------------------------------
# 8. Export Processed Data
# ---------------------------------------------------------
save(
  sweden_shape,
  combined_data,
  sweden_map_data,
  file = here::here("data", "processed_sweden.RData")
)
message("Processed data saved to data/processed_sweden.RData")

Visualization script

3_visualization.R:

  1. Sources libraries and preprocessed data.

  2. Formats percentage labels (e.g., 12.3%).

  3. Builds an interactive tmap–Leaflet map of direct estimates and saves it as self-contained HTML.

  6. Saves both map objects for reproducibility or further customization.

# ---------------------------------------------------------
# 3_visualization.R
# ---------------------------------------------------------

# ---------------------------------------------------------
# 1. Source Libraries and Data

library(here)

# ---------------------------------------------------------
# 2. Prepare Map Data
# ---------------------------------------------------------
# Create formatted labels for percent estimates
sweden_map_data <- sweden_map_data %>%
  mutate(
    Percent_label = scales::label_percent(accuracy = 0.1)(Percent_2025K1)
  )

SAE script

4_SAE.R:

  1. Sources required libraries and data.

  2. Visualizes direct estimates to inspect trends.

  3. Builds spatial weight matrix from county polygons to test for spatial correlation in the direct estimates.

  4. Prepares data frame for emdi, specifying sampling variances.

  5. Fits a series of Fay–Herriot models:

    • Initial (REML) with all covariates.

    • Stepwise (ML) for model selection via KICb2.

    • Transformed versions using arcsine–BC.

  6. Maps each model’s estimates (and MSE/CV) via emdi::map_plot and ggplot2, exporting PNGs.

  7. Generates diagnostic and comparison plots (direct vs. FH).

  8. Extracts coefficient tables, formats with gt, and saves grouped model comparison tables.

extract_fh <- function(x){
  sm <- summary(x)
  m  <- as.matrix(sm$model$coefficients)
  df <- as.data.frame(m, stringsAsFactors=FALSE)
  df$term <- rownames(df)
  df <- df %>% dplyr::rename(
    estimate  = coefficients,
    std.error = std.error,
    t.value   = t.value,
    p.value   = p.value
  ) %>% dplyr::select(term, estimate, std.error, t.value, p.value)
  df
}
